# Denet - Сервис управления пользователями и задачами

## Описание проекта

Denet - это REST API сервис для управления пользователями и их задачами. Сервис предоставляет функционал для:
- Управления пользователями (создание, чтение, обновление, удаление)
- Выполнения задач пользователями
- Реферальной системы
- Получения списка пользователей с сортировкой и фильтрацией
- Получения топа пользователей по очкам

## Технологии

- Язык: Go 1.21+
- Фреймворк: Chi (роутинг)
- База данных: PostgreSQL
- Аутентификация: JWT (RSA)
- Логирование: Zerolog
- Миграции: sql-migrate
- Тестирование: testify, go test
- Контейнеризация: Docker

## Установка и запуск

### Требования
- Установленный Go 1.21+
- Установленный Docker и Docker Compose (для запуска в контейнерах)
- PostgreSQL

### Запуск локально

1. Клонировать репозиторий:
```bash
git clone https://github.com/romanpitatelev/denet.git
cd denet
```

2. Установить зависимости
```
make tidy
```

3. Запустить PostgreSQL (через Docker)
```
make up
```

4. Запустить приложение
```
make run
```

5. Собрать в Docker
```
make image
```

## Конфигурация

Настройки приложения задаются через переменные окружения (см. example.env):\
LOG_LEVEL - уровень логирования (по умолчанию debug)\
BIND_ADDRESS - адрес и порт для сервера (по умолчанию :8081)\
POSTGRES_DSN - строка подключения к PostgreSQL

## API Endpoints

Все запросы требуют JWT токена в заголовке Authorization: Bearer <token>

### Пользователи

Получить информацию о пользователе\
GET /api/v1/users/{id}/status

Обновить информацию о пользователе\
PATCH /api/v1/users/{id}\
Тело запроса:
{
  "name": "string",
  "email": "string",
  "role": "string"
}

Удалить пользователя\
DELETE /api/v1/users/{id}

Получить список пользователей\
GET /api/v1/users\
Параметры запроса:
- sorting - поле для сортировки (name, points)
- descending - сортировка по убыванию (true/false)
- limit - количество записей
- offset - смещение
- filter - строка для поиска

Получить топ пользователей
GET /api/v1/users/leaderboard

### Задачи

Выполнить задачу\
POST /api/v1/users/{id}/task/complete\
Тело запроса:
{
  "type": "telegram|twitter"
}

Выполнить реферальную задачу\
POST /api/v1/users/{id}/referrer\
Тело запроса:
{
  "userReferenceId": "uuid"
}
